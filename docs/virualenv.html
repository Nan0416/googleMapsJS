1. Create a new environment. (Do not root privilege.)

    virtualenv [--python=/usr/bin/python2.7] envName

  2. Enter this env directory, create a file called requirements.txt to tell what package you need.

    Create a file called requirements.txt. In this file, we declare the packages and version numbers.

    package_name==package_version

    django==1.9.0

  3. Start this virtual environment.

    source ./bin/activate

    Principle: this "activate" file is actually a shell script. To create a new python environment, virtualenv just simply copies the python source file into a new location. And when we run this activate, it will change the PS1 (the prompt) and the search path of python.

  4. Download dependencies

    Attention: this must be download after activate the new enviroment.

    pip install -r requirements.txt

How to manually install package. -- Introduction to pip.

  1. Where does python manage dependencies?

    Most dependencies are stored in ../usr/lib/pythonVer/site-packages/ [native python] or envDir/lib/pythonVer/site-packages/ [virtualenv created env].

  2. Download source code.

    There are two ways to download source files. 1. Download from github or the official website. Compared with the second approach, this is a safer way. Second, using pip download, which download the newest version of the source file. e.g. pip download django.

  3. Install source code

    Uncompress source file into the site-packages directory. Go to the source file directories, run python setup.py install
