Main configuration

  1. Structure of files

    site-avaiable VS site-enabled. mods_avaiable VS mods_enabled

    file in the *enabled directories will be load automatically

    Making an alias from the avaiable directories to enabled directories is the prefect way to cite mods and site(host)

  2. Include other configuration file

    Include /usr/local/apache2/conf/ssl.conf

    Include /etc/apache2/site-availabe/defualt.conf

  3. KeepAlive

    This allows a presistent TCP connection that allows multiple http requests.

    One TCP connection is also called a HTTP session.

  4. LoadModule module_name module_path

  5. <Directory path></Directory>

    directive inside this part will only work on this path filesystem.

    • AllowOveride None // this is for security when path is "/" in order to denied the .htaccess

  6. <VirtualHost *:80></VirtualHost>

    Add a vhost

    • DocumentRoot path
    • ServerName www.example.com
    • ServerAlias *.example2.com
    • RewriteEngine On
    • Order deny, allow // order tell the apache the step of applying deny and allow, in this case, will allow from all.
    • Deny from 192.0.0.1 // deny to server the user with 192.0.0.1
    • Allow from all
